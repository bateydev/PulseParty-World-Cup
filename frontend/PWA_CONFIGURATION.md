# PWA Configuration Documentation

## Overview

PulseParty Rooms is configured as a Progressive Web Application (PWA) with offline support, installability, and optimized caching strategies.

**Requirements Addressed:**
- 6.1: PWA with web app manifest enabling installation on mobile devices
- 6.2: Service worker that caches static assets for offline access
- 6.3: App icon on user's home screen when installed
- 6.4: Display cached content and indicate offline status when network is lost
- 6.5: Synchronize pending actions and resume real-time updates when connectivity is restored

## Architecture

### 1. Vite PWA Plugin Configuration

**File:** `vite.config.ts`

The PWA is configured using `vite-plugin-pwa` with the following key settings:

- **Register Type:** `autoUpdate` - Automatically updates the service worker when a new version is available
- **Manifest:** Comprehensive web app manifest with app metadata and icons
- **Workbox:** Advanced caching strategies for different resource types

### 2. Web App Manifest

**Generated by:** `vite-plugin-pwa`
**Static reference:** `public/manifest.webmanifest`

Key properties:
- **name:** "PulseParty Rooms"
- **short_name:** "PulseParty"
- **theme_color:** `#1e40af` (blue-800)
- **background_color:** `#ffffff`
- **display:** `standalone` (full-screen app experience)
- **orientation:** `portrait` (optimized for mobile)

### 3. Service Worker Caching Strategies

#### Cache-First Strategy (Static Assets)

Used for images and fonts that rarely change:

```typescript
{
  urlPattern: /^https?:.*\.(png|jpg|jpeg|svg|gif|webp|ico)$/,
  handler: 'CacheFirst',
  cacheName: 'images-cache',
  expiration: {
    maxEntries: 100,
    maxAgeSeconds: 2592000, // 30 days
  }
}
```

**Benefits:**
- Instant loading of images and icons
- Reduced bandwidth usage
- Works completely offline

#### Network-First Strategy (API Responses)

Used for API calls with cache fallback:

```typescript
{
  urlPattern: /^https:\/\/.*\.execute-api\..*\.amazonaws\.com\/.*/,
  handler: 'NetworkFirst',
  cacheName: 'api-cache',
  networkTimeoutSeconds: 10,
  expiration: {
    maxEntries: 50,
    maxAgeSeconds: 300, // 5 minutes
  }
}
```

**Benefits:**
- Always tries to fetch fresh data first
- Falls back to cache if network is unavailable
- 10-second timeout prevents long waits

#### Network-Only Strategy (WebSocket)

Used for real-time WebSocket connections:

```typescript
{
  urlPattern: /^wss?:\/\/.*/,
  handler: 'NetworkOnly',
}
```

**Benefits:**
- Real-time data is never cached
- Ensures users always get live updates when online

### 4. PWA Registration Module

**File:** `src/pwa-registration.ts`

Provides:
- Service worker lifecycle management
- Online/offline detection
- Update notifications
- Cache management utilities

**Key Functions:**

```typescript
// Initialize PWA features
initializePWA()

// Subscribe to status changes
onPWAStatusChange((status) => {
  console.log('Online:', status.isOnline);
  console.log('Installed:', status.isInstalled);
  console.log('Needs Update:', status.needsUpdate);
})

// Activate pending update
activateUpdate()

// Clear all caches (debugging)
clearAllCaches()

// Check if running as installed PWA
isStandalone()
```

### 5. PWA Status Component

**File:** `src/components/PWAStatus.tsx`

React component that displays:
- **Offline Indicator:** Yellow banner when network is unavailable
- **Update Notification:** Blue card when new version is available
- **Reconnection Indicator:** Green banner (auto-dismisses) when back online

**Usage:**

```tsx
import { PWAStatusIndicator } from './components/PWAStatus';

function App() {
  return (
    <>
      <PWAStatusIndicator />
      {/* Rest of app */}
    </>
  );
}
```

**Hook for accessing status:**

```tsx
import { usePWAStatus } from './components/PWAStatus';

function MyComponent() {
  const { isOnline, needsUpdate } = usePWAStatus();
  
  return (
    <div>
      {isOnline ? 'Connected' : 'Offline'}
    </div>
  );
}
```

## Offline Behavior

### What Works Offline

✅ **Cached UI:** All static assets (HTML, CSS, JS, images, fonts)
✅ **Last Known State:** Previously loaded match data and leaderboards
✅ **Queued Actions:** Prediction submissions are queued for sync

### What Doesn't Work Offline

❌ **Real-time Updates:** WebSocket connections require network
❌ **New Data:** Cannot fetch new match events or room data
❌ **Room Creation/Joining:** Requires API calls

### Offline Indicator

When offline, users see:
- Yellow banner at top: "You're offline. Some features may be limited."
- Cached content continues to display
- Interactive elements are disabled or show pending state

### Reconnection

When network is restored:
- Green banner briefly appears: "Back online"
- WebSocket reconnects automatically (with exponential backoff)
- Queued actions are synchronized
- Real-time updates resume

## Troubleshooting

### Service Worker Not Updating

**Solution:**
1. Open DevTools → Application → Service Workers
2. Check "Update on reload"
3. Click "Unregister" and reload
4. Or use `clearAllCaches()` function

### Icons Not Displaying

**Solution:**
1. Ensure PNG icons exist in `public/` directory
2. Generate from SVG: `convert -background none public/pwa-icon.svg -resize 192x192 public/pwa-192x192.png`
3. Clear browser cache and reload

### Offline Mode Not Working

**Solution:**
1. Check service worker is registered: DevTools → Application → Service Workers
2. Verify cache storage: DevTools → Application → Cache Storage
3. Ensure HTTPS (service workers require secure context)
4. Check console for errors

### Update Not Showing

**Solution:**
1. Service worker updates check every 60 seconds
2. Force update: DevTools → Application → Service Workers → "Update"
3. Or reload page (with "Update on reload" checked)
